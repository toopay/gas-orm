Gas ORM

A lighweight and easy-to-use ORM for CodeIgniter

Download Download fork Gas ORM on GitHub

Overview

A lighweight and easy-to-use Active Record ORM for CodeIgniter. Gas was built specifically for CodeIgniter app. It uses standard CI DB packages, also take anvantages from CI validator class. Gas ORM provide a set of methods that will map your database tables and its relationship, into accesible object.

Features

  • Auto-create models from database tables and vice versa.
  • Cascading insert, update, delete across entities (composite key supported).
  • Supported databases : cubrid, mssql, mysql, oci8, odbc, postgre, sqlite3, sqlsrv, PDO.
  • Support multiple database connection.
  • Support modular models directories.
  • Multiple relationship with custom relationship setting.
  • Per-request caching.
  • Eager Loading, to maximize your relationship queries.
  • Various finder method (can chained with most of CI AR) and aggregates.
  • Built-in validation and auto-mapping input collection, with minimal setup.
  • Hooks points, to control over your model.
  • Extensions, to share your common function/library across your model
  • Transaction and other CI AR goodness.

Overview

// FINDER
$all = Model\User::all();
$some_user = Model\User::find_by_username('foo', FALSE);
$first_user = Model\User::first();

// WRITE (INSERT, UPDATE, DELETE) AND VALIDATION
$new_user = Model\User::make($_POST)->save(TRUE);
$new_user->name = "New Name";
$new_user->save();
$new_user->delete();

// RELATIONSHIP & EAGER LOADING
$some_user = Model\User::with('wife', 'kid', 'job')->all();
$some_user_wife = $some_user->wife;
